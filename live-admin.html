Payment & Store Settings

📢 Note for Global Configuration Sync:
Settings updated below will only apply to your local browser session. To publish these settings (UPI details, payment keys, or Firebase configuration) live to all customers globally, please copy the configuration parameters and save them in the settings.json file in your codebase and redeploy the site.

UPI QR & App Payment Gateway

If enabled, checkout routing will redirect to the custom UPI payment gateway page.

Leave empty to automatically generate a dynamic, pre-filled QR code matching the order price.

Firebase Real-Time Database

If enabled, product inventory will read/write from/to Firestore instead of local storage.

Local Database Backups

If the Firebase database is offline or daily free limits are hit, you can save your changes locally by exporting them as JSON files. Place the files in your project directory (as products.json and settings.json) and deploy the code.
